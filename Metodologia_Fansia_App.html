⚡ Diccionario del Vibecoder

10 prompts universales que te sacan de cualquier apuro. Tenlos siempre a mano.

1. Generar código base

"Eres un desarrollador senior experto en [Python/Pandas/LangChain]. Genera un script que [descripción precisa]. Incluye: 1) logging, 2) manejo de errores con try/except específicos, 3) type hints, 4) comentarios en cada bloque explicando QUÉ hace y POR QUÉ. El script debe ser ejecutable tal cual."

2. Depurar error

"Este es mi código [pegarlo] y este el error [pegarlo]. Dame 3 posibles causas ordenadas por probabilidad. Para cada causa: 1) por qué ocurre, 2) cómo verificarlo, 3) código corregido. Si la causa 1 no funciona, ¿qué más puede ser?"

3. Explicar código

"Explica este código [pegarlo] como si tuviera 15 años y está aprendiendo a programar. No uses jerga técnica sin explicarla. Para cada bloque, dime: 1) qué hace, 2) por qué es necesario, 3) qué pasaría si lo quitamos. Termina con un resumen de 3 oraciones."

4. Refactorizar

"Refactoriza este código [pegarlo]. Hazlo más legible, modular y mantenible. Separa en funciones, añade type hints, sigue PEP8. No cambies la funcionalidad. Dame el código completo refactorizado más una lista de qué cambiaste y por qué."

5. Añadir feature

"A este código [pegarlo] quiero añadirle: [feature]. No reescribas todo el código, solo muéstrame los cambios necesarios: 1) qué líneas añadir, 2) dónde, 3) qué import añadir. Explica por qué tu solución funciona."

6. Mejorar prompts

"Eres un experto en prompt engineering. Ayúdame a mejorar este prompt: [mi prompt]. Hazlo más específico, ponle restricciones, formato de salida esperado y ejemplos few-shot. Dame 3 versiones mejoradas con explicación de cada mejora."

7. Escribir tests

"Genera tests unitarios con pytest para este código [pegarlo]. Cubre: 1) caso normal, 2) caso borde (empty, None, valores extremos), 3) caso de error esperado. Los tests deben ser ejecutables con 'pytest test_*.py'. Usa assert."

8. Comparar librerías

"Compara [librería A] vs [librería B] para: [tarea]. Dame: 1) pros/cons en lenguaje de calle, 2) cuándo usar cada una, 3) ejemplo mínimo funcional de cada una para la misma tarea, 4) mi recomendación."

9. Convertir código entre frameworks

"Convierte este código [pegarlo] de [librería A] a [librería B]. Mantén exactamente la misma funcionalidad. Explica las diferencias clave de paradigma entre ambos."

10. Revisión de seguridad

"Revisa este código [pegarlo] en busca de: 1) vulnerabilidades de seguridad (inyección, exposición de API keys, path traversal), 2) malas prácticas, 3) problemas de performance. Para cada hallazgo: riesgo (Alto/Medio/Bajo) y cómo arreglarlo."

🛡️ Plan de Contingencia

3 atajos para cuando te estanques. No pierdas tiempo.

⚡ Atajo 1: "Si no entiendes ML, sáltate la teoría"

  • No entiendes qué es un árbol de decisión? → No importa.
  • Solo memoriza el pipeline: import → split → fit → predict → score
  • Random Forest funciona sin tuning. No toques hiperparámetros aún.
  • Problema: Accuracy baja → solución: agrega más datos, no cambies el modelo.

⚡ Atajo 2: "Si un error te para por +30 min, copia y pega"

  • Copia el error completo en ChatGPT o Claude: "Explica este error de Python como si tuviera 12 años y dame el código corregido"
  • Si usas Copilot en VS Code, selecciona el error y presiona Ctrl+I → "Fix this"
  • Regla de oro: No pierdas más de 30 min en un error. Pídele a la IA que lo resuelva.

⚡ Atajo 3: "Si LangChain te abruma, hazlo sin LangChain"

  • Puedes hacer RAG con 20 líneas de Python puro usando openai.Embedding.create() + numpy para similitud coseno.
  • LangChain es comodidad, no necesidad. Si no entiendes una clase, hazlo manual.
  • Fórmula mágica: similitud = np.dot(emb1, emb2) / (np.linalg.norm(emb1) * np.linalg.norm(emb2))

📉 Plan B si te atrasas

  • Semana 1-2 atrasada? → Elimina el proyecto de API, ve directo a ML
  • Semana 5-6 atrasada? → Salta ChromaDB, usa FAISS (más fácil)
  • Nunca te saltes el proyecto final. Si llegas justo, haz el proyecto final aunque sea solo el RAG sin Streamlit.

🔄 Flujo de Depuración Asistida (15 min)

  • 0-5 min: Intentas resolverlo tú (lees el error, buscas el typo)
  • 5-10 min: Le pides a la IA: "Este es el error [pegarlo], dime 3 posibles causas y cómo solucionarlas"
  • 10-15 min: Ataque nuclear: "Reescribe esta función desde cero usando un enfoque más simple. Evita [librería compleja], usa solo [librería simple]"
  • Si a los 15 min no funciona: Borra todo y empieza de nuevo con un prompt diferente.
